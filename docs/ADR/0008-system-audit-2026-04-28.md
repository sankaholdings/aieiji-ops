# ADR-0008: 全環境整合性監査と是正方針（2026-04-28夜）

- **Auditor**: Claude（自宅Fujitsu D: セッション）

- **Triggered by**: 社長指摘「ただシステムを増やすだけでなく、退役・修正を含めた総合判断を」

## Context（背景）

ADR-0007 までで AIEiji 関連システムは**機能追加が一巡**した。しかし社長から：

> 全環境（ローカル / GitHub / 1106PC / memory / claude.ai）を**総合的に判断**し、退役すべきは退役、修正すべきは修正で対応する指示を出したが、その遂行内容は確認できているのか

という指摘あり。本ADRは ADR-0004（Reality Check Protocol）の延長として、全環境の整合性を監査した結果と、それに基づく是正方針を記録する。

## Decision（決定）

### D1: 1106PC 役割の正式記述を memory に追加
新規 memory `project_1106pc_active_services.md` を作成し、現役2タスク（Orchestrator + GmailThreadWatcher）を明文化。`project_1106pc_identity.md` も更新。

### D2: 段階的退役（Issue化して順次実施）

「即座に全削除」ではなく、**Issue化→Archive移動→1〜2週間の様子見→本削除**の3段階運用（`project_archive_convention.md` 準拠）。

退役対象を**5グループ**に分類し、それぞれ Issue を1本起票：

- **Issue #22**: ルート直下の廃止アーティファクト退役（HANDOVER / STATUS / session_log / 旧 .ps1 / docs/PC_RESPONSIBILITY 等）
- **Issue #23**: 廃止GASパイプライン痕跡退役（00_Inbox / Scripts/GAS_Chatwork / 04_SE 配下）
- **Issue #24**: memory廃止済記述の退役（handoff_to_*.md / project_dual_pc_sync / project_sync_blockers / reference_status_md）
- **Issue #25**: 1106PC bridge.ps1 等の死蔵スクリプト整理（Phase B0残骸 / process_inbox.py / collect_mf_journals.py の用途確認・退役判断）
- **Issue #26**: 廃止済パス参照の腐敗修正（`ClaudeSync` 等を grep して各文書を更新 or 退役）

### D3: `.claude-memory/` ルート直下コピーの取り扱い決定

選択肢：
- (a) 削除：active memoryのみを正本とし、PC間同期は諦める（ADR-0001準拠）
- (b) 維持：3PC共有の参照用として残す（ただし更新時の二重管理コスト）

→ **(a) 削除** とする。memoryはPC毎独立、ADR-0001の精神を守る。一時的な参照は GitHub の ADR で代替可能。

### D4: 退役運用の標準化

`project_archive_convention.md` を本ADRのリンクで補強し、以下を恒久ルールとする：

- 退役は `_Archive_YYYY-MM-DD/` へ移動（即削除しない）
- 移動から **30日後** に本削除（カレンダー or Issueで自動リマインド検討）
- 各退役は Issue でログ化（commit historyだけでは追跡しにくいため）

### D5: Out of Scope（今回はやらない）

- 02_CFO/ の中身精査（まだCFO業務で使われている可能性あり、別途棚卸し）
- 06_SE/aieiji_secretary_v2/ の HANDOFF_*.md 等の旧引継書（v3作業の記録として一時残す）
- ADRディレクトリ自体の整理（ADR-0001〜0008 全て生きており不要）
