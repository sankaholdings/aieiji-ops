# 設計原則（DESIGN_PRINCIPLES）

> **本ドキュメントは全システム設計の前提条件である。**
> 新規開発・既存改修の際、必ず本原則に照らして検討すること。
> 違反する設計は採用しない。

---

## 原則1：中国モード前提（最重要）

### 前提
社長は **月の約1/3を中国で過ごす**。これは定常的な業務環境の一部であり、例外ではない。
「日本でのみ動くシステム」は不完全とみなす。

### 中国でブロックされるもの

| サービス | 状況 |
|---|---|
| Claude.ai (Anthropic) | 完全ブロック |
| GitHub | 不安定（頻繁にブロック） |
| Gmail / Google Calendar / Drive | 完全ブロック |
| ChatGPT / OpenAI | 完全ブロック |
| WhatsApp / Telegram | ブロック |

### 第一防衛線：良之助VPN
社長は「良之助」VPNを長年使用。通常時は日本と同等の環境で作業可能。
**ただし完全に信頼してはならない。**

### VPN失効の現実シナリオ

| シナリオ | 頻度 | 影響期間 |
|---|---|---|
| 党大会等の政治的イベントによる強化遮断 | 年数回 | 数日〜数週間 |
| Claude.aiの異常アクセス判定によるロック | たまに | 数時間〜数日 |
| Google系の追加認証要求 | 頻繁 | 毎回手間 |
| Tailscale + VPN競合 | 設定次第 | 1106PC接続不可 |

### Plan A / Plan B 二段構え（必須）

すべての重要システムは以下を持たねばならない：

- **Plan A（通常）**: 良之助VPN → 日本側システム
- **Plan B（VPN死亡時）**: 中国国内サービスのみで完結
  - **Qwen（通义千问）** ← Claude代替
  - **Chatwork** ← 通知・連絡（基本動く）
  - **1106PC**（Chatwork経由トリガー） ← 日中橋渡し中継点
  - **Gitee** ← GitHub代替（必要に応じ）

### 設計時の必須チェック

- [ ] このシステムは中国でVPN経由で動くか？
- [ ] VPNが2-3日落ちたとき、社長は何ができなくなるか？
- [ ] そのとき代替経路でカバーできるか？
- [ ] 必要な情報は中国側にも持ち出せる形になっているか？
- [ ] 日本側と中国側の橋渡し経路は確保されているか？

### 1106PCの戦略的位置づけ

1106PCは日本に常駐する「**日中の橋渡し中継点**」。
- Chatwork経由の指示は中国からも届く
- 1106PCが日本側のGitHub/Gmail/Claudeを叩いて結果をChatworkで返す
- VPN死亡時の最後の砦

最も堅牢な経路は **Chatwork → 1106PC** の片方向トリガー。

## 原則2：3PC環境パリティ

### 前提
社長は3台のPCを使い分ける：

| PC | 役割 | ドライブ |
|---|---|---|
| 自宅Fujitsu（ノート、中国持参） | サブ作業端末・**中国出張時のメイン** | D: |
| 職場PC | メイン作業端末 | E: |
| 1106PC（mini PC） | 24h自律実行サーバー | G: |

### 「会話の同期」は断念済み

3PC間の「会話継続」は2026-04-24に断念。
会話継続はClaude.ai Projects（クラウド）が解決する。

### 「環境の一致」は維持する

ただし、各PCの**作業環境**は揃えなければならない：

| 対象 | 同期方法 |
|---|---|
| Claude Code memoryファイル | GitHub経由（手動 or 自動） |
| 設定ファイル（settings.json等） | GitHub経由 |
| スクリプト・ツール | aieiji-ops リポジトリ |
| MCPコネクタ設定 | Claude.ai側で一元管理（PCに依存しない） |

### 設計時の必須チェック

- [ ] このシステムは3PC全てで動くか？
- [ ] 特定PCにしかないファイル・設定に依存していないか？
- [ ] 自宅PCを中国に持参した状態でも機能するか？
- [ ] 1106PCの自律実行で必要な情報は全てリポジトリにあるか？

### 「あるPCでは動くが別では動かない」を許容しない

例：「この処理は1106PCでしか動かない」「自宅PCには設定がないから無理」
こうした断片化は3PC運用を破綻させる。GitHub経由で必ず揃える。

## How to apply

- **新規開発時**：両原則を満たすかを設計レビューで確認
- **既存改修時**：両原則への影響を確認
- **Issue処理時**：完了コメントに「中国モード対応：YES/NO」「3PC対応：YES/NO」を明記
- **違反提案を避ける**：「日本でしか動かない」「特定PCでしか動かない」前提の実装は不採用

## Why this exists

過去の設計（AIEijiチーム / GASパイプライン / 旧3PC同期構想）は両原則を満たしていなかった。
結果として中国出張時に使えない、特定PCに縛られるなどの問題が発生した。
**最初から両原則を前提に設計する方が、結果的にシンプルで堅牢になる。**
